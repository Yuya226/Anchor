# Anchor SPEC

## 目的

Anchor は、自分専用の iOS アプリである。

目的は、迷いやすい自分が、今やることに戻れる「アンカー」を持つこと。

このアプリは、完璧な人生設計をするためのアプリではない。
選択肢が多すぎて動けなくなる状態を防ぎ、迷い・実行・無行動・棚上げを記録し、自分の内省を外部化・体系化するためのアプリである。

## 設計思想

### 1. 迷いを否定しない

迷いは失敗ではない。
迷いは、選択肢の多さと向き合った記録である。

ただし、迷い続けて行動が止まる場合は、それを見える化し、扱える形にする。

### 2. 迷いは解決せず、まず預ける

実行中に迷ったら、その場で結論を出そうとしない。
迷いログに保存し、今やることに戻る。

### 3. 選ばなかった選択肢は捨てずに棚上げする

「切る」は、可能性を否定することではない。
今この瞬間に迷い続けるコストを減らすため、一時的に選択肢を閉じることである。

### 4. 価値は、やりきった後に見える

やる前に意味を完全に証明しなくてよい。
一定時間取り組んだ後に、直接成果と副産物の両方を回収する。

### 5. 最初は自分専用

競合調査、市場調査、ユーザー拡大、収益化は考えない。
まずは自分が毎日使える最小版を作る。

## MVPの範囲

最初のMVPでは、以下だけを作る。

1. 今日のアンカー画面
2. 迷いログ入力
3. 迷いログ一覧
4. 棚上げリスト
5. 作業タイマー
6. チェックイン

## MVPで作らないもの

- カレンダー
- 通知
- ウィジェット
- iCloud同期
- AI分析
- 複雑なグラフ
- ログイン
- 複数ユーザー対応
- App Store公開
- 収益化機能

## 画面構成

### 1. ホーム画面 / 今日のアンカー

毎日最初に見る画面。

表示するもの：

- 今日のアンカー
- 今の重点テーマ
- 次の一手
- 今日の積み上げ時間
- 今日の迷い回数
- 棚上げ中の項目
- 作業開始ボタン
- 迷ったボタン
- チェックインボタン

役割：

迷ったときに戻る場所。
今やることを思い出す画面。

### 2. 迷いログ画面

迷った内容を保存・確認する画面。

保存するもの：

- 迷った内容
- 作成日時
- 関連テーマ
- 迷いの強さ
- 結果

最初は content と createdAt だけでよい。

最初の一覧画面は閲覧専用でよい。
削除、編集、検索、分類、強さの記録は後回しにし、まずは「預けた迷いを見返せる」ことを優先する。

### 3. 棚上げリスト画面

今はやらないが、捨てない選択肢を保存する画面。

保存するもの：

- 棚上げしたこと
- 理由
- 作成日時
- 見直し日

### 4. 作業タイマー画面

何にどれだけ時間を投下したかを記録する画面。

保存するもの：

- 作業名
- 開始時刻
- 終了時刻
- 作業時間

最初はホーム画面内の簡易タイマーでもよい。

### 5. チェックイン画面

1日の終わりに軽く記録する画面。

保存するもの：

- 達成度 1〜5
- 納得度 1〜5
- 迷いの強さ 1〜5
- 一言メモ
- 作成日時

## データモデル案

### DoubtLog

迷いログ。

Fields:

- id: UUID
- content: String
- createdAt: Date

将来的に追加する可能性：

- intensity: Int
- relatedTheme: String
- result: String

### ShelfItem

棚上げ項目。

Fields:

- id: UUID
- title: String
- reason: String
- createdAt: Date
- reviewDate: Date?

### CheckIn

日次チェックイン。

Fields:

- id: UUID
- achievementScore: Int
- satisfactionScore: Int
- doubtScore: Int
- memo: String
- createdAt: Date

### WorkSession

作業時間ログ。

Fields:

- id: UUID
- title: String
- startedAt: Date
- endedAt: Date?
- durationSeconds: Int

## デザイン方針

- 落ち着いた雰囲気
- 自分を責めない言葉遣い
- 余白を多めにする
- カードUI中心
- 情報を詰め込みすぎない
- 1画面で「戻ってこられる感覚」を出す
- 派手な色や過度なアニメーションは不要

## 文言方針

避ける表現：

- 達成率が低いです
- もっと頑張りましょう
- 未完了タスクがあります
- 失敗しました

使いたい表現：

- 迷いはここに預ける
- 次の一手に戻る
- 今日は迷いが多い日でした
- 記録すれば、次に使えます
- 今は考えるより進める時間です
- 棚上げして、今やることに戻る

## 実装順

1. 静的ホーム画面
2. DoubtLog モデル作成
3. 迷ったボタンから sheet で迷い入力
4. 今日の迷い回数をホームに反映
5. 迷いログ一覧画面
6. ShelfItem モデル作成
7. 棚上げリスト追加・一覧
8. WorkSession モデル作成
9. 作業開始・停止・積み上げ時間表示
10. CheckIn モデル作成
11. チェックイン入力画面
12. 簡単な履歴表示

## Codexへの基本方針

- 変更範囲は小さくする
- 一度に多くの機能を実装しない
- 既存のUIを大きく崩さない
- まずビルドが通ることを優先する
- 外部ライブラリは使わない
- SwiftDataで保存する
- 複雑な抽象化はまだしない
- まず自分が使える最小実装を優先する

## 現在の優先順位

最優先は、以下の体験を作ること。

迷ったときにアプリを開く  
→ 今日のアンカーを見る  
→ 迷いをログに預ける  
→ 今やることに戻る  

この体験が成立するまでは、他の機能を増やしすぎない。
