# Anchor SPEC v3

## 0. 現在の状態

Anchor は SwiftUI + SwiftData で作る、自分専用の iOS アプリである。

MVP の初期実装は一通り完了している。

実装済みの最小機能：

1. 今日のアンカー画面
2. 迷いログ入力
3. 迷いログ一覧
4. 棚上げリスト
5. ActivitySession
6. 作業開始 / 停止
7. セッション終了後フィードバック
8. 代替行動ログ
9. チェックイン
10. 簡単な履歴表示

ここからは、MVPを壊さずに、次の機能候補を順番に消化していく。

---

## 1. アプリの目的

Anchor は、自分の内省を外部化・体系化するためのアプリである。

目的は、迷いやすい自分が、今やることに戻れる「アンカー」を持つこと。

このアプリは、完璧な人生設計をするためのアプリではない。

選択肢が多すぎて動けなくなる状態を防ぎ、迷い・実行・無行動・棚上げ・代替行動・回復を記録し、自分に合った実行スタイルを見つけていくためのアプリである。

---

## 2. 中心思想

### 2.1 迷いを否定しない

迷いは失敗ではない。

迷いは、選択肢の多さと向き合った記録である。

ただし、迷い続けて行動が止まる場合は、それを見える化し、扱える形にする。

### 2.2 迷いは解決せず、まず預ける

実行中に迷ったら、その場で結論を出そうとしない。

まず迷いログに保存し、今やることに戻る。

### 2.3 選ばなかった選択肢は捨てずに棚上げする

「切る」は、可能性を否定することではない。

今この瞬間に迷い続けるコストを減らすため、一時的に選択肢を閉じることである。

棚上げは、捨てることではなく、後で見直すための保管である。

### 2.4 価値は、やりきった後に見える

やる前に意味を完全に証明しなくてよい。

一定時間取り組んだ後に、直接成果と副産物の両方を回収する。

### 2.5 毎日コツコツを唯一の正解にしない

Anchor は、毎日コツコツできない自分を矯正するアプリではない。

自分に合った実行リズムを記録し、再現しやすくするアプリである。

Minimum / Daily / Momentum / Hybrid / Recovery を扱う。

### 2.6 実行前は理想に寄せ、実行後は現実を受け止める

実行前の画面では、選択肢を増やさない。

ホーム画面には、本当に今すべきことを中心に表示する。

娯楽・逃避・代替行動は、実行前の主要選択肢としては並べすぎない。

一方で、実行後には、本来のタスクができたのか、別のことをしたのか、何もしなかったのかを記録できるようにする。

### 2.7 Anchor は現代版の日記である

Anchor は、長文を書く日記アプリではない。

本来やる予定だったこと、実際にやったこと、迷ったこと、棚上げしたこと、何もしなかったこと、代わりに流れた行動を、選択・短文・時間ログによって軽く記録する現代版の日記である。

## 5. 実行スタイル方針

Anchor は、タスクやテーマをすべて同じ扱いにしない。

タスクには、適した実行スタイルがある。

### 5.1 Minimum

やる気がない日でも、最低限これだけできたらよい行動。

目的は、大きく進めることではなく、接続を切らさないこと。

「何もしない日」を減らし、次に戻ってくる足場を残すためのもの。

例：

- Anchorを開いて今日のアンカーを見る
- 次の一手だけ確認する
- SPECを1分読む
- ニュース見出しを1つ見る
- 英単語を3個見る
- 机に座る
- アプリを起動する

Minimum は、成果を出すためのタスクというより、再開のためのタスクである。

### 5.2 Daily

短時間で毎日続けたい行動。

目的は、生活の基礎を整えたり、継続的に触れることで感覚を維持すること。

例：

- ニュースを見る
- 英単語を見る
- 軽いストレッチ
- 今日の予定を確認する
- 軽いチェックイン
- 日経や金融ニュースの見出し確認

Daily は、重くしすぎない。

毎日続けたいが、できなかった日を過度に責める対象にはしない。

### 5.3 Momentum

発火したときに、一気に大きく進める行動。

目的は、まとまった時間と集中力を使って大きく前進すること。

例：

- アプリ開発
- レポート作成
- 資格勉強の詰め込み
- 試験前の集中学習
- プログラミング学習
- 大きめの読書や研究

Momentum は、意志だけで毎日任意に発動できるものではない。

外部要因、締切、危機感、初速の成功体験、強い興味などによって発火することがある。

Anchor は、Momentum を毎日発動させるアプリではない。

Momentum が発生しやすい条件を観察し、発生したときに逃さず乗り、後から成果と条件を記録する。

### 5.4 Hybrid

普段は軽く接続し、発火したら一気に進める行動。

Minimum / Daily と Momentum の中間にある。

例：

- 資格勉強
- 語学
- プログラミング学習
- 研究
- 読書
- キャリア準備

Hybrid では、普段は5分だけ触る、資料を開くだけ、次の一手を確認するだけでもよい。

Momentum が来たときには、まとまった時間を使って一気に進める。

### 5.5 Recovery

疲労・落ち込み・燃え尽き時に、次に戻るための回復行動。

Recovery はサボりではない。

心身を戻し、Minimum / Daily / Hybrid / Momentum に戻るための準備である。

例：

- 寝る
- 散歩する
- 風呂に入る
- 部屋を片付ける
- スマホを置く
- 軽く食べる
- 何も考えず休む
- チェックインだけする
- 今日はMinimumだけにする

Minimum は前進との接続を切らさないためのもの。

Recovery は心身を戻すためのもの。

## 6. Momentum と点火トリガー

Momentum は直接オンにできるものではない。

ただし、発火確率を上げる条件は設計できる可能性がある。

点火トリガーの例：

- 締切を作る
- 人に宣言する
- 最初の1画面だけ作る
- ChatGPTと壁打ちする
- Codexに小さい実装を任せる
- GitHubに初回pushする
- 模試やテストを入れる
- 損失が発生する条件を作る
- 徹夜しても翌日回復できる日を確保する

点火トリガーは、必ず発火するボタンではない。

発火確率を上げる装置として扱う。

Momentum が発生したときは、後から再現できるように記録する。

記録したい項目：

- 何がきっかけだったか
- いつ始まったか
- どれくらい続いたか
- 何が進んだか
- なぜ止まらなかったか
- 途中で迷いはあったか
- 翌日の疲労はどうだったか
- 回復できたか
- また使いたい戦略か

徹夜や長時間作業は、単純に悪いものとして扱わない。

ただし、高負荷スプリントとして、成果と反動の両方を記録する。

## 7. タスクとコンテキスト

タスクは、常に同じ条件で実行できるわけではない。

実行可能性は、今の状況によって変わる。

例：

- 家にいる
- 移動中
- バイトの休憩中
- 授業中の空き時間
- スマホしか使えない
- PCが使える
- 5分だけある
- 30分以上ある
- 集中できる
- 疲れている

将来的には、タスクに実行コンテキストを持たせる。

例：

- 必要デバイス：スマホ / PC / どちらでも可
- 実行場所：家 / 外 / 移動中 / 授業中 / 休憩中
- 必要時間：5分 / 15分 / 30分 / 60分
- 必要集中度：低 / 中 / 高

これにより、今の状況で実行可能なタスクだけを出せるようにする。

## 8. 主要画面

### 8.1 ホーム画面 / 今日のアンカー

毎日最初に見る画面。

役割：

- 迷ったときに戻る場所
- 今やることを思い出す場所
- 本当に今すべきことだけを見る場所

表示したいもの：

- 今日のアンカー
- 今の重点テーマ
- 次の一手
- 今日の積み上げ時間
- 今日の迷い回数
- 棚上げ中の項目
- Minimum / Daily / Momentum / Hybrid / Recovery の状態
- 作業開始ボタン
- 迷ったボタン
- チェックインボタン

### 8.2 迷いログ画面

迷った内容を保存・確認する画面。

保存するもの：

- 迷った内容
- 作成日時
- 関連テーマ
- 迷いの強さ
- 結果

最初は content と createdAt だけでよい。

一覧画面はまず閲覧できればよい。

編集、検索、分類、強さの記録は後回しにしてもよい。

### 8.3 棚上げリスト画面

今はやらないが、捨てない選択肢を保存する画面。

保存するもの：

- 棚上げしたこと
- 理由
- 作成日時
- 見直し日

### 8.4 ActivitySession画面

作業・行動時間を記録する画面。

Anchor が記録するのは、勉強や作業だけではない。

実際に起きた行動全体である。

### 8.5 セッション終了後フィードバック画面

作業開始後、終了または中断したときに、軽いフィードバック画面を表示する。

基本は重くしない。

最初の選択肢：

- できた
- 少しできた
- 別のことをした
- 何もしなかった

「別のことをした」を選んだ場合、代替行動を選べるようにする。

代替行動の例：

- YouTube
- Netflix
- アニメ
- 映画
- 筋トレ
- 散歩
- 睡眠
- ポルノ
- 別の勉強
- バイト
- 何もしない
- その他

一言メモは任意。

作業終了時は「事実ログ」に寄せる。

深い意味づけは夜チェックイン側に寄せる。

### 8.6 チェックイン画面

1日の終わりに軽く記録する画面。

役割：

- その日の意味づけ
- 現代版の日記
- 作業ログでは拾えない感覚の記録

保存するもの：

- 達成度 1〜5
- 納得度 1〜5
- 迷いの強さ 1〜5
- 一言メモ
- 作成日時

将来的には、今日の状態タグを追加してもよい。

例：

- 迷っていた
- 疲れていた
- なんとなく流れた
- 気分転換したかった
- 思ったより進んだ
- 集中できた
- モメンタムが来た
- 回復日だった

### 8.7 履歴画面

過去の迷い、棚上げ、ActivitySession、チェックインを振り返る画面。

目的は、分析というより、まず「自分の記録を見返せる」こと。

- 落ち着いた雰囲気

- ホームでは「次の一手」を最も見えやすくする
- 入力画面は重くしない
- 終了後フィードバックは数タップで終わるようにする

### 10.1 避ける表現

- 達成率が低いです
- もっと頑張りましょう
- 未完了タスクがあります
- 失敗しました

### 10.2 使いたい表現

- 迷いはここに預ける
- 次の一手に戻る
- 今日は迷いが多い日でした
- 記録すれば、次に使えます
- 今は考えるより進める時間です
- 棚上げして、今やることに戻る
- 今日はMinimumだけで接続維持
- Momentumが来ています
- 今日はRecoveryに寄せてもいい日です

## 14. 現在の最優先

次の最優先は、Task / Theme 管理である。

理由：

今のMVPは、迷い・棚上げ・記録はできる。

しかし、ホームに表示される「今本当にすべきこと」がまだ固定または弱い。

Anchorの中心体験は、

迷ったときにアプリを開く  
→ 今日のアンカーを見る  
→ 今本当にすべきことを見る  
→ 迷いを預ける  
→ 実行する  
→ 実際に何が起きたか記録する  

である。

この体験を完成させるために、次は Task / Theme を導入する。
